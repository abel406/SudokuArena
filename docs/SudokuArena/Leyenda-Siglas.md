# Leyenda de Siglas (SudokuArena)

Actualizado: 2026-02-25

## Regla
- Toda sigla nueva usada en documentacion tecnica debe agregarse aqui.
- Si una sigla deja de usarse, debe removerse de esta leyenda en el mismo cambio.

## Siglas de Features/Backlog
- `AC`: AutoComplete (autocompletado asistido).
- `BM`: Board Modes (modos de tablero).
- `DS`: Dataset y Solver propio.
- `GS`: Game Start (inicio de partida y selector de dificultad).
- `PC`: Puzzle Catalog (catalogo runtime de puzzles/provider).
- `PD`: Puzzle Data Pipeline (pipeline de datos).
- `UI`: User Interface (interfaz de usuario).

## Siglas Tecnicas Generales
- `AB`: A/B testing.
- `API`: Application Programming Interface.
- `APK`: Android Package Kit.
- `CI`: Continuous Integration.
- `DAO`: Data Access Object.
- `DB`: Database (base de datos).
- `DI`: Dependency Injection.
- `DTO`: Data Transfer Object.
- `E2E`: End-to-End testing.
- `ELO`: sistema de rating competitivo.
- `IoC`: Inversion of Control.
- `JWT`: JSON Web Token.
- `LAN`: Local Area Network.
- `MVP`: Minimum Viable Product.
- `SDK`: Software Development Kit.
- `SE` (en `weighted_se`): Score Estimate (estimacion de score/dificultad del solver).
- `UI`: User Interface.
- `UX`: User Experience.
- `VM`: ViewModel.
- `WPF`: Windows Presentation Foundation.

## Convenciones de Dataset
- `QB` (en nombres como `defaultQb*`): Question Bank.
- `qid`: Question ID (identificador de puzzle).
- `time_map`: mapa de umbrales de tiempo por puzzle.
- `solver_details`: metricas de dificultad por puzzle.
